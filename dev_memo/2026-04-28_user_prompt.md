## 2026-04-28 — 初期依頼

wbsに起票されたタスクから、outlookのスケジュールを作成する仕組みを作って欲しいです。背景は、wbsとoutlookとが連動しないことで、wbsタスクが後回しになることが多く、連動させることで、処理漏れを回避できるためです。逆に、outlookからwbsに反映させるルートも欲しいです。wbsのフォーマットを置いておきます。WBS_Sample.xlsxです。タスクはidで管理して、変化があれば更新するようにしてください。ここに記入する私のプロンプトは、開発メモフォルダを作り残しておいてください。ユーザーは、ダウンロードするidと反映させるidを指定できるようにして下さい。idはこの場合、Microsoft 365と紐づくメールアドレスが入っているAP_MailAddress欄の値になります。ガントチャート表示のための日付は変動するので、新規作成時にユーザーから範囲を受け取れると良いかと思います。アウトルックのスケジュールをどうやって作るのか、逆にどうやってダウンロードしてデータするのかなど、いろいろ工夫が必要そうです。質問あれば日本語で教えて下さい。

## 2026-04-28 — 質問への回答

1. Microsoft 365 / Outlook接続方法 → **ローカルOutlook (COM自動化)**
2. カレンダーイベントの日程 → **StartDate_p / FinishDate_p（計画日）**、時間指定。Excelに時間フィールドを追加
3. Outlook → WBS の取り込み → **TaskIDを件名に埋め込んでマッチング**、StartDate/FinishDateに反映
4. AP_MailAddress が None の行 → **サンプル値は無視。ブランクに記入してもらう**
5. 実行方法 → **CLI（当初）**
